125 /* Dean Gaudet's algorithm. To be most useful there must be a good way
126 to evaluate the C "conditional expression" (a?b:c construction) without
127 branching. The result of a?b:c is b if a is true (nonzero), and c if a
128 is false (0).
129  For example, a compare to zero op that sets a target GPR to 1 if the
130 operand is 0, and to 0 if the operand is nonzero, will do it. With this
131 instruction, the algorithm is entirely branch-free. But the most
132 interesting thing about it is the high degree of parallelism. All six
133 lines with conditional expressions can be executed in parallel (on a
134 machine with sufficient computational units).
135  Although the instruction count is 30 measured statically, it could
136 execute in only 10 cycles on a machine with sufficient parallelism.
137  The first two uses of y can instead be x, which would increase the
138 useful parallelism on most machines (the assignments to y, bz, and b4
139 could then all run in parallel). */
